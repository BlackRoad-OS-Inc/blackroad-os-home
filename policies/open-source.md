# Open Source Policy

## Using Open Source

- Prefer well-maintained, widely adopted libraries with permissive licenses (MIT, Apache 2.0, BSD).
- Avoid copyleft licenses (e.g., GPL) in proprietary components unless approved by the orchestrator.
- Record dependencies and licenses to support compliance checks.

## License Compatibility & Attribution

- Ensure third-party licenses are compatible with BlackRoad’s distribution model.
- Preserve copyright notices and attribution where required.
- Include license files when redistributing or embedding OSS components.

## Contributing Back

- Contributions to external projects require orchestrator approval when tied to core IP or security-sensitive code.
- Follow upstream contribution guidelines and disclose affiliations transparently.
- Journal significant contributions for auditability and knowledge sharing.
