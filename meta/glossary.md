# Glossary

- **Orchestrator**: The single human decision-maker (Cecilia) who directs agents and provides consent.
- **Agent**: An automated worker operating within a defined domain and policy set.
- **Capability**: A function or skill an agent can perform to deliver outcomes.
- **Task**: A unit of work assigned to an agent or pod.
- **Event**: A signal or input that triggers agent processing.
- **JournalEntry**: A structured record in `blackroad-os-core` capturing decisions, rationale, and outcomes.
- **PS-SHA∞**: The journaling and hashing mechanism used for immutable decision logs.
- **Prism Console**: The interface for monitoring agents, journals, and system health.
- **OS Core**: The domain models and schemas defining BlackRoad’s primitives (see `blackroad-os-core`).
- **Operator**: The agent and orchestrator runtime that executes tasks (see `blackroad-os-operator`).
- **Pod**: A group of agents operating within a domain under shared goals and controls.
- **Runbook**: Procedural guidance for operations, often stored in `blackroad-os-infra`.
