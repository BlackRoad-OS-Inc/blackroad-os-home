# Contributing

## How to propose changes

- Open a pull request with a clear summary, rationale, and links to related issues or journal entries.
- Keep PRs focused on a small set of files to simplify review.
- Update checklists and cross-references when adding new policies or playbooks.

## Review requirements

- At least one reviewer should approve changes to handbook sections.
- The orchestrator must approve changes to policies and any material governance updates.
- Large reorganizations should include a brief migration note describing how to map old content to new locations.

## General expectations

- Preserve clarity and brevity; prioritize readability for both humans and agents.
- Avoid massive unreviewed changes to policies.
- Ensure citations or references to other repos remain accurate.
