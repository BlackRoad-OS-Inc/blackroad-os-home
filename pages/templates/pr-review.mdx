---
title: PR Review Template
description: Checklist for thorough reviews and release notes.
---

## Pre-flight

- Linked issue or decision log entry.
- Screenshots or recordings for UI changes.
- Tests that matter are referenced (unit, e2e, linters).

## Checklist

- [ ] Scope matches the described change.
- [ ] Naming and API signatures are clear.
- [ ] Error handling and logging are present.
- [ ] Security and privacy considerations are addressed.
- [ ] Docs updated (handbook or runbook links).

## Release notes

Use the following template:

```
## Summary
- What changed and why

## Impact
- User-facing changes
- Ops/SLO impact

## Rollout
- Deployment plan
- Verification steps
```

## Follow-up

- Track TODOs in the issue tracker.
- Link post-incident actions back to this PR when relevant.
